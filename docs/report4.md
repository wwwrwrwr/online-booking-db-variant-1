## Цель работы
Реализовать главную бизнес-операцию системы — создание и управление записями клиентов на услуги.

---

## Реализованные страницы

### 1. Список записей
**URL:** `?entity=appointment&action=list`
- Таблица всех записей с цветовой индикацией статусов
- Зелёный — запланировано, красный — отменено, серый — проведено
- Фильтры по дате и статусу
- Кнопки просмотра и отмены

### 2. Создание записи
**URL:** `?entity=appointment&action=create`
- Выпадающие списки клиента, врача и услуги
- Выбор даты и времени
- Проверка свободного слота перед сохранением
- Защита от двойного бронирования

### 3. Просмотр записи
**URL:** `?entity=appointment&action=view&id=N`
- Полная информация о записи
- Данные клиента, врача и услуги
- Кнопка отмены для запланированных записей

### 4. Отчёты
**URL:** `?entity=appointment&action=reports`
- Отчёт 1: количество записей и выручка по дням
- Отчёт 2: рейтинг врачей по количеству записей
- Отчёт 3: количество отменённых записей

---

## Бизнес-логика

### Защита от двойного бронирования
Перед созданием записи выполняется проверка:
```sql
SELECT COUNT(*) FROM appointments
WHERE dentist_id = :dentist_id
AND appointment_datetime = :datetime
```
Если слот занят — запись не создаётся.

### Управление статусами
Отменить можно только запланированную запись. Проведённые и уже отменённые записи изменить нельзя.
